# 灵山剑侠录 · 童年回忆版 开发计划

> 给 Claude Code 用的项目计划。目标:复刻经典 2D 仙剑的体验感,核心是「进屋找宝贝」。
> 基线代码:`lingshan-rpg.html`(单文件,~44KB,无外部依赖)

## 现状基线(v1 已完成)

- 960×640 像素画布,卷轴大地图(64×48 村庄+森林+断桥)+ 锁妖塔室内地图(24×30)
- 手绘像素角色 12 个(16×20 ~ 24×26 格),两帧行走动画
- 回合制战斗:五灵相克(水火雷风土)、仙术、物品、装备、防御计算、Boss 施法 AI
- 商店×2、客栈、存档(localStorage)、标题画面、BGM/音效(WebAudio)
- 已有自动化冒烟测试(node + DOM stub)和视觉回归(node-canvas 截图)流程

## 愿景与边界

**愿景**:还原小时候玩 2D 仙剑的感觉——进每一户人家、翻柜子水缸、"叮"一声找到宝贝、主人站在旁边毫无意见。

**非目标(本版不做)**:高清重绘全部立绘(已决定保持现有像素风)、3D、联机、移动端原生 App。

## Phase 1 · 进屋找宝贝(P0,核心交付)

**机制设计**
- 室内地图系统:村里 4 间房各做一张小室内图(约 12×10),门口瓦片 `D` 双向传送(现有 `MAPS` registry + `switchMap()` 直接支持,无需改架构)
- 可调查点(POI):每张室内图配置 `POIS[mapName] = [{x, y, kind, loot, flagId}]`,kind 含:柜子、水缸、米缸、书架、床底、坛子、灶台
- 交互方式:复用现有"撞击触发"模式(同 `npcAt()`),走向家具按方向键即调查
- 宝物表:银两(10–50 两)、回灵丹、清心散、偶尔空手("什么都没有……"也是情怀的一部分)、1–2 个剧情彩蛋纸条
- 反馈:找到宝物播"叮"上扬音效 + 金色飘字;已翻过的点提示"已经翻过了"
- 存档:`save()` 增加 `looted: {flagId: true}`,翻过的点跨存档持久

**验收标准**
- [x] 4 间房都能进出,室内有家具陈设和主人 NPC(冒烟测试覆盖,含门口通路可达性)
- [x] 每间房 ≥3 个可调查点,全村合计 ≥12 个(实际 3+4+4+4=15 个)
- [x] 找到宝物有音效 + 飘字;重复调查有不同文案(每种家具各有重复文案)
- [x] 翻东西时主人有吐槽对话彩蛋(翻到第二件时触发,每屋一次)
- [x] 存档正确记录已翻过的点;手机浏览器可玩(撞击交互与 dpad 同路径;真机回归待人工)

## 技术备忘(给 Claude Code 的提示)

- `MAPS` 是地图注册表,`cur`/`curName` 指向当前地图;`switchMap(name, x, y)` 切图,室内图直接往里加
- 瓦片字符约定:`G` 草 `t` 深草 `T` 树 `W` 水 `P` 路 `b` 桥 `R` 屋顶 `H` 墙 `X` 石墙 `F` 石地 `e` 符纹遇敌地 `D` 门 `O` 出口 `S` 灵泉 `B` 封印——室内新增建议:`f` 木地板、家具用 POI 层而非瓦片层
- 像素角色是字符串像素图(`PIX` + 调色板 `PALS`),新家具贴图照同格式加
- 数值层:`S` 玩家、`INV` 背包、`EQ` 装备、`flags` 剧情位、`save()/loadSave()` 存档
- 测试不能丢:改完逻辑跑冒烟测试,改完画面跑截图对比
